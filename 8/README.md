# Модуль 2, задание 8: статическая трансляция портов

Настройка выполняется вручную на `HQ-RTR` и `BR-RTR`.

## Параметры

Перед настройкой определите актуальные адреса:

| Параметр | Пример |
|---|---|
| Адрес HQ-SRV | `192.168.100.2` |
| Внешний адрес HQ-RTR со стороны ISP | `172.16.1.2` |
| Адрес BR-SRV | `192.168.0.2` |
| Внешний адрес BR-RTR со стороны ISP | `172.16.2.2` |

Если адреса в варианте отличаются, замените их во всех командах.

Используемые порты:

| Назначение | Внешний порт | Порт сервера |
|---|---:|---:|
| Apache на HQ-SRV | `8080` | `80` |
| Docker-приложение на BR-SRV | `8080` | `8080` |
| SSH на HQ-SRV | `2026` | `2026` |
| SSH на BR-SRV | `2026` | `2026` |

## HQ-RTR

```text
enable
configure terminal
ip nat source static tcp 192.168.100.2 80 172.16.1.2 8080
ip nat source static tcp 192.168.100.2 2026 172.16.1.2 2026
exit
write memory
```

В результате:

- `172.16.1.2:8080` перенаправляется на `192.168.100.2:80`;
- `172.16.1.2:2026` перенаправляется на `192.168.100.2:2026`.

## BR-RTR

```text
enable
configure terminal
ip nat source static tcp 192.168.0.2 8080 172.16.2.2 8080
ip nat source static tcp 192.168.0.2 2026 172.16.2.2 2026
exit
write memory
```

В результате:

- `172.16.2.2:8080` перенаправляется на `192.168.0.2:8080`;
- `172.16.2.2:2026` перенаправляется на `192.168.0.2:2026`.

## Проверка с ISP

Проверка веб-приложения на `HQ-SRV`:

```bash
curl http://172.16.1.2:8080/ | head
```

Проверка Docker-приложения на `BR-SRV`:

```bash
curl http://172.16.2.2:8080/ | head
```

Проверка SSH:

```bash
ssh -p 2026 sshuser@172.16.1.2
ssh -p 2026 sshuser@172.16.2.2
```

Также сайты должны открываться во внешней сети по адресам:

```text
http://172.16.1.2:8080/
http://172.16.2.2:8080/
```

Если соединение не устанавливается, сначала проверьте доступность сервисов
непосредственно на серверах:

```bash
# HQ-SRV
ss -lntp | grep ':80'
ss -lntp | grep ':2026'

# BR-SRV
ss -lntp | grep ':8080'
ss -lntp | grep ':2026'
```
