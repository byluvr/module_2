# Модуль 2, задание 2: RAID-хранилище

Скрипт создаёт программный RAID на `HQ-SRV`, файловую систему ext4 и
автоматическое монтирование в `/raid`.

## Важно

Создание массива уничтожает данные и таблицы разделов на дисках из
`RAID_DISKS`. Скрипт `00-list-disks.sh` автоматически выбирает свободные
дополнительные диски указанного размера, но не разрешает их очистку.

Перед запуском войдите как `root` и просмотрите устройства:

```bash
chmod +x ./*.sh
./00-list-disks.sh
```

Скрипт:

1. Создаёт `.env` из `.env.example`, если файл отсутствует.
2. Читает `EXPECTED_DISK_SIZE_GIB` и `RAID_LEVEL`.
3. Исключает системный диск и устройства с точками монтирования.
4. Находит целые неразмеченные диски нужного размера.
5. Записывает найденные устройства в `RAID_DISKS` внутри `.env`.
6. Сбрасывает `ERASE_DISKS=no`, чтобы новый список нельзя было стереть без
   просмотра.

Например:

```bash
RAID_DISKS="/dev/sdb /dev/sdc"
```

Системный диск и диски с точками монтирования автоматически исключаются.
Перед очисткой всё равно проверьте строку `RAID_DISKS`.

## Настройка

Создайте рабочий файл параметров:

```bash
cp .env.example .env
chmod 600 .env
```

Основные параметры:

```bash
RAID_NAME=md0
RAID_LEVEL=0
RAID_DISKS=""
CREATE_PARTITION=yes
MOUNT_POINT=/raid
ERASE_DISKS=no
```

Поддерживаются уровни RAID `0`, `1` и `5`. Количество устройств берётся из
`RAID_DISKS` автоматически. RAID 5 требует минимум три диска.

`RAID_NAME` можно заменить на `md1` и другое имя формата `mdN`.

## Раздел или файловая система на массиве

По умолчанию `CREATE_PARTITION=yes`, потому что в задании требуется создать
раздел. Будут созданы:

```text
/dev/md0
└─/dev/md0p1  ext4  /raid
```

Некоторые методички создают ext4 непосредственно на `/dev/md0`. Для такого
варианта укажите:

```bash
CREATE_PARTITION=no
```

## Запуск

Сначала автоматически определите диски:

```bash
./00-list-disks.sh
```

После проверки записанной строки `RAID_DISKS` разрешите очистку:

```bash
sed -i 's/^ERASE_DISKS=no/ERASE_DISKS=yes/' .env
./01-hq-srv-raid.sh
```

Флаг `ERASE_DISKS=yes` обязателен только для создания разделов, массива и
файловой системы. Уже настроенный массив повторный запуск не форматирует.

Скрипт:

1. Проверяет устройства, размер и отсутствие точек монтирования.
2. Устанавливает `mdadm`.
3. Очищает старые сигнатуры на выбранных дисках.
4. Создаёт RAID нужного уровня.
5. Обновляет `/etc/mdadm.conf` без дублирования строки `ARRAY`.
6. Создаёт один GPT-раздел, если включён `CREATE_PARTITION`.
7. Создаёт ext4.
8. Записывает в `/etc/fstab` UUID файловой системы.
9. Монтирует хранилище в `/raid`.

Резервные копии изменяемых системных файлов сохраняются в
`/root/module_2_task_2_backups`.

## Проверка

```bash
cat /proc/mdstat
mdadm --detail /dev/md0
lsblk -f
findmnt /raid
df -hT /raid
grep /raid /etc/fstab
grep '^ARRAY /dev/md0' /etc/mdadm.conf
```

Для RAID 0 в выводе `mdadm --detail` ожидается:

```text
Raid Level : raid0
Raid Devices : 2
```
