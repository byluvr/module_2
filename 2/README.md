# Модуль 2, задание 2: RAID-хранилище

Скрипт создаёт программный RAID на `HQ-SRV`, файловую систему ext4 и
автоматическое монтирование в `/raid`.

## Важно

Создание или изменение массива уничтожает данные на его дисках. Скрипт
`00-list-disks.sh` автоматически формирует `RAID_DISKS`, но не меняет
`ERASE_DISKS`.

Перед запуском войдите как `root` и просмотрите устройства:

```bash
chmod +x ./*.sh
./00-list-disks.sh
```

Скрипт:

1. Создаёт `.env` из `.env.example`, если файл отсутствует.
2. Читает `EXPECTED_DISK_SIZE_GIB` и `RAID_LEVEL`.
3. Исключает системный диск и устройства с точками монтирования.
4. Сохраняет текущих участников массива `RAID_NAME`.
5. Добавляет свободные неразмеченные диски нужного размера.
6. Записывает устройства в `RAID_DISKS` внутри `.env`.

Значение `ERASE_DISKS` остаётся таким, каким вы задали его до запуска.

Например:

```bash
RAID_DISKS="/dev/sdb /dev/sdc"
```

Системный диск и диски с точками монтирования автоматически исключаются.
Перед очисткой всё равно проверьте строку `RAID_DISKS`.

## Настройка

Создайте рабочий файл параметров:

```bash
cp .env.example .env
chmod 600 .env
```

Основные параметры:

```bash
RAID_NAME=md0
RAID_LEVEL=0
RAID_DISKS=""
CREATE_PARTITION=yes
MOUNT_POINT=/raid
ERASE_DISKS=no
```

Поддерживаются уровни RAID `0`, `1` и `5`. Количество устройств берётся из
`RAID_DISKS` автоматически. RAID 5 требует минимум три диска.

`RAID_NAME` можно заменить на `md1` и другое имя формата `mdN`.

## Раздел или файловая система на массиве

По умолчанию `CREATE_PARTITION=yes`, потому что в задании требуется создать
раздел. Будут созданы:

```text
/dev/md0
└─/dev/md0p1  ext4  /raid
```

Некоторые методички создают ext4 непосредственно на `/dev/md0`. Для такого
варианта укажите:

```bash
CREATE_PARTITION=no
```

## Запуск

Сначала отредактируйте параметры `.env`. Для экзаменационного RAID 0:

```bash
RAID_NAME=md0
RAID_LEVEL=0
EXPECTED_DISK_SIZE_GIB=1
CREATE_PARTITION=yes
ERASE_DISKS=yes
```

Затем запускайте скрипты подряд:

```bash
./00-list-disks.sh
./01-hq-srv-raid.sh
```

`00-list-disks.sh` заполнит `RAID_DISKS`, сохранив `ERASE_DISKS=yes`.

`01-hq-srv-raid.sh` работает как приведение к состоянию из `.env`:

- если уровень, состав дисков и схема разделов совпадают, массив и данные
  сохраняются;
- если изменён уровень RAID, количество или состав дисков, массив
  размонтируется и пересоздаётся;
- если изменён `CREATE_PARTITION`, хранилище также пересоздаётся;
- пересоздание разрешено только при `ERASE_DISKS=yes`.

Например, для перехода на RAID 5:

```bash
sed -i 's/^RAID_LEVEL=.*/RAID_LEVEL=5/' .env
./00-list-disks.sh
./01-hq-srv-raid.sh
```

`00` сохранит текущие диски массива и добавит свободные диски размером
`EXPECTED_DISK_SIZE_GIB`. RAID 5 требует минимум три устройства.

Важно: переход RAID 0 → RAID 5 в этом решении выполняется пересозданием,
поэтому существующие данные в `/raid` будут удалены.

Скрипт:

1. Проверяет устройства и их размер.
2. Устанавливает `mdadm`.
3. Сравнивает текущий массив с параметрами `.env`.
4. При расхождении размонтирует и останавливает старый массив.
5. Очищает старые сигнатуры и создаёт RAID нужного уровня.
6. Обновляет `/etc/mdadm.conf` без дублирования строки `ARRAY`.
7. Создаёт один GPT-раздел, если включён `CREATE_PARTITION`.
8. Создаёт ext4.
9. Записывает в `/etc/fstab` UUID файловой системы.
10. Монтирует хранилище в `/raid`.

Резервные копии изменяемых системных файлов сохраняются в
`/root/module_2_task_2_backups`.

## Проверка

```bash
cat /proc/mdstat
mdadm --detail /dev/md0
lsblk -f
findmnt /raid
df -hT /raid
grep /raid /etc/fstab
grep '^ARRAY /dev/md0' /etc/mdadm.conf
```

Для RAID 0 в выводе `mdadm --detail` ожидается:

```text
Raid Level : raid0
Raid Devices : 2
```
